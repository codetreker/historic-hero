# HH-VIS-001: 三国人物关系图谱可视化 — PRD

日期：2026-04-28 | 状态：Draft（v2 更新：建军反馈后补充性能、数据、搜索需求）

---

## 背景

建军希望构建一个中国历史人物关系可视化产品，首个版本聚焦三国时期（184-280 年）。目标是让用户通过交互式图谱，直观探索三国人物之间的关系网络、了解重要历史事件，兼具"随便逛逛"的探索乐趣和"深入了解"的知识深度。

数据初版已完成：439 位人物、433 条关系、95 个历史事件，数据来源为维基百科（均标注来源 URL）。**注意：当前数据量不足，需大幅补充（见需求 8）。**

---

## 目标用户

- **历史爱好者**：对三国历史有兴趣，想系统了解人物关系
- **学生/研究者**：需要梳理三国人物脉络、查阅人物生平
- **休闲浏览者**：好奇驱动，想随便点点看，发现有趣的历史关联

---

## 核心需求

### 需求 1: 关系图谱（主视图）

- **用户故事**：作为历史爱好者，我想看到一张三国人物关系网络图，以便直观理解各势力之间的人物关系
- **验收标准**：
  - [ ] 打开页面即可看到人物关系图谱，人物为节点，关系为连线
  - [ ] 不同阵营（魏/蜀/吴/东汉/西晋/其他）用不同颜色区分，颜色含义有图例说明
  - [ ] 核心人物（关系数量多的）节点更大，边缘人物节点较小
  - [ ] 关系连线上显示关系类型标签（如"君臣""父子""对手"）
  - [ ] 支持鼠标拖拽移动视图、滚轮缩放、拖拽节点
  - [ ] 500+ 节点规模下，交互流畅无明显卡顿
  - [ ] **不能一次性渲染全量节点**——必须采用渐进式/动态加载策略，默认只渲染当前视口或当前过滤条件下的节点
  - [ ] 首次打开不白屏、不卡顿（建军实测当前版本浏览器卡死）

### 需求 2: 分层浏览（防信息过载）

- **用户故事**：作为用户，我不想一次看到 400+ 个节点的混乱图谱，我想从大局到细节逐步探索
- **验收标准**：
  - [ ] 默认视图按阵营分组聚合，先看到 6 个势力集团（魏/蜀/吴/东汉/西晋/其他）
  - [ ] 点击某个阵营 → 展开该阵营内部人物关系
  - [ ] 可以一键回到阵营总览视图
  - [ ] 展开某阵营时，与其他阵营的跨阵营关系仍然可见

### 需求 3: 搜索

- **用户故事**：作为用户，我想搜索某个我感兴趣的人物，直接定位到他
- **验收标准**：
  - [ ] 页面顶部有搜索框，支持按人物姓名、字、号搜索
  - [ ] 支持拼音输入搜索（输入 "caocao" 能匹配到曹操）
  - [ ] 输入时实时显示匹配建议列表（模糊匹配），按相关度排序（完全匹配 > 前缀匹配 > 包含匹配）
  - [ ] 建议列表中显示人物名 + 阵营 + 身份，方便区分同名人物
  - [ ] 选择某人物后，图谱自动居中到该人物，并高亮其直接关联的人物和关系网络
  - [ ] 搜索结果为空时有友好提示（如："未找到匹配人物，试试其他关键词"）

### 需求 4: 过滤

- **用户故事**：作为用户，我想只看某个阵营或某种身份的人物，减少干扰
- **验收标准**：
  - [ ] 提供阵营过滤：魏/蜀/吴/东汉/西晋/其他，可多选
  - [ ] 提供身份过滤：帝王/武将/谋士/文臣/诸侯/文人/后妃/宦官等
  - [ ] 提供关系类型过滤：君臣/父子/兄弟/夫妻/对手/结义/背叛/师徒等
  - [ ] 过滤后图谱实时更新，只显示符合条件的节点和关系
  - [ ] 过滤条件可组合使用（如：蜀汉 + 武将）
  - [ ] 可一键清除所有过滤条件

### 需求 5: 人物详情面板

- **用户故事**：作为用户，我想点击某个人物，看到他的详细信息
- **验收标准**：
  - [ ] 点击图谱中的人物节点，右侧弹出详情面板
  - [ ] 详情面板包含：姓名、字、封号/头衔、阵营、身份、生卒年、籍贯
  - [ ] 详情面板包含：人物简介（一段话描述生平）
  - [ ] 详情面板包含：关系列表（所有与此人物相关的关系，可点击跳转到关联人物）
  - [ ] 详情面板包含：参与事件列表（可点击查看事件详情）
  - [ ] 详情面板包含：数据来源链接（维基百科 URL，可点击跳转）
  - [ ] 面板可关闭

### 需求 6: 事件详情

- **用户故事**：作为用户，我想了解三国时期的重要历史事件，以及哪些人物参与了
- **验收标准**：
  - [ ] 从人物详情面板的"参与事件"可进入事件详情
  - [ ] 事件详情包含：事件名称、时间（年份）、地点、类型（战役/政治/外交…）
  - [ ] 事件详情包含：事件描述
  - [ ] 事件详情包含：参与人物列表（含各人物在该事件中的角色），可点击跳转回人物
  - [ ] 事件详情包含：事件结果
  - [ ] 事件详情包含：数据来源链接

### 需求 7: 时间轴导航

- **用户故事**：作为用户，我想按时间线浏览三国历史，看到不同时期的关键事件和活跃人物
- **验收标准**：
  - [ ] 页面顶部或底部有朝代/时间轴导航条（约 184-280 年）
  - [ ] 时间轴上标注重要事件节点
  - [ ] 点击或拖动时间轴可筛选特定时间段内活跃的人物（根据生卒年判断）
  - [ ] 时间轴与图谱联动：选择某时间段后，图谱只显示该时段活跃的人物和发生的事件

### 需求 8: 数据完整性（blocking）

- **用户故事**：作为历史爱好者，我期望数据足够丰富，人物关系网络密集、事件覆盖全面，而不是稀疏零散的
- **验收标准**：
  - [ ] 人物总数 ≥ 500（当前 439，需补充）
  - [ ] 关系总数 ≥ 1500 条（当前 433 条，每人平均不到 1 条关系，远远不够）
  - [ ] 核心人物（曹操/刘备/孙权/诸葛亮/关羽/张飞/司马懿等）每人 ≥ 10 条关系
  - [ ] 历史事件 ≥ 200 个（当前 95 个，需补充）
  - [ ] 每个事件至少关联 3 个人物
  - [ ] 数据必须从互联网公开来源获取（维基百科等），标注来源 URL，不使用 AI 凭空生成
- **优先级**：P0（blocking）——数据不够丰富，图谱没有看头

## 非功能需求

- **语言**：中文界面，人物名/事件名/UI 全部中文
- **性能**：500+ 节点规模下交互响应 < 500ms，不出现明显卡顿或白屏。**必须采用动态加载/渐进渲染策略，禁止一次性加载全量数据到 DOM/Canvas**
- **浏览器**：支持主流现代浏览器（Chrome/Edge/Firefox/Safari 最新版）
- **响应式**：桌面端优先，但在平板上基本可用
- **无需登录**：纯公开访问，无账号体系
- **纯静态部署**：无后端依赖，数据内置在前端包中

---

## 不在范围

- ❌ 其他朝代（秦、汉、唐、宋…）——后续扩展，本次只做三国
- ❌ 用户贡献/编辑数据——只读产品
- ❌ 多语言支持——只做中文
- ❌ 移动端优化——桌面优先
- ❌ 后端 API / 数据库——纯静态
- ❌ 用户账号/收藏/标注功能
- ❌ 数据导出功能
